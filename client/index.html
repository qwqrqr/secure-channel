Secure Channel - учебный стенд

Дипломный проект: Разработка защиты корпаративных данных

Запуск стенда

  1. В терминале: cd gateway && npm start && npm run static
  2. Откройте этот index.html в браузере (через Live Server )
  3. Откройте две вкладки: одну как HQ, другую как Remote
  4. В обеих вкладках укажите одинаковый ID туннеля и нажмите «Подключиться»
  5. Стороны автоматически проведут ECDH-handshake → канал готов
  6. Отправляйте сообщения, открывайте «Перехватчик» для демонстрации

Что демонстрировать на защите

  1. Установка туннеля. Показать handshake в журналах обеих сторон, обратить внимание на одинаковый отпечаток сеансового ключа.
  2. Передача сообщения. Отправить нормальный текст, показать что он пришёл расшифрованным.
  3. Перехват. Открыть Eve - там видно только base64 шифротекст, никакого открытого текста.
  4. Атака подмены. Нажать «Атака: подмена пакета» - на той стороне HMAC не сойдётся, пакет отвергнут с пометкой «АТАКА ПЕРЕХВАЧЕНА».
  5. DLP. Попробовать отправить «Мой ИИН 123456789012» - DLP заблокирует ещё до шифрования.
  6. Anti-replay. Через журнал шлюза показать поле counter и пояснить, что повторная отправка того же пакета отвергается.
